## üõ†Ô∏è Python ETL Pipeline

**The data underwent a rigorous** **Extraction, Transformation, and Loading (ETL)** **process using Pandas and SQLAlchemy.**

### 1. Cleaning & Normalization

* **Text Standardisation:** **Applied** **unicodedata** **to remove accents, special characters (**@**), and converted all text to lowercase with underscores for SQL compatibility.**
* **Data Imputation:**

  * **Categorical nulls (Region, Product) were filled using the** **Mode**.
  * **Numerical nulls (Price, Discount) were filled using the** **Mean**.
  * **Logic Reconciliation:** **The** **total** **column was recalculated using the formula:** **(Quantity * Unit_Price) - Discount + Shipping_Cost**.
* **Feature Engineering:** **Created derived columns:** **year**, **month**, **total_ventas**, and **segmento_cliente** **(calculated via quantiles).**

### 2. Loading

**Used** **SQLAlchemy** **to establish a secure connection via** **.env** **variables, automating the migration of 4 normalized tables into the** **Assessment** **database.**

## üóÑÔ∏è Database Architecture: Star Schema

 **To optimize query performance and reporting, the flat data was normalized into a** **Star Schema**:

* **Fact Table:** **fact_ventas** **(Contains IDs, Quantity, and Revenue metrics).**
* **Dimension Tables:**

  * **dim_productos**: Product names, types, and unit prices.
  * **dim_regiones**: Geographic data (City, Country).
  * **dim_clientes**: Customer types and segments.

## üîç SQL Analysis

**The following intermediate and advanced queries were developed to extract business value:**

### 1. Top 5 Products by Revenue

**code**SQL

```
SELECT p.producto, SUM(v.total_ventas) as revenue
FROM fact_ventas v
JOIN dim_productos p ON v.id_producto = p.id_producto
GROUP BY 1 ORDER BY 2 DESC LIMIT 5;
```

* **Purpose:** **Identifying hero products for stock priority.**

### 2. Year-over-Year (YoY) Comparison

**code**SQL

```
SELECT year, total_sales,
LAG(total_sales) OVER (ORDER BY year) as last_year,
((total_sales - LAG(total_sales) OVER (ORDER BY year)) / LAG(total_sales) OVER (ORDER BY year)) * 100 as yoy_growth
FROM annual_sales_summary;
```

* **Purpose:** **Measuring annual growth momentum.**

**Used to classify customers based on total spend without skipping ranks in case of ties.**
